Файл — именованная область данных на носителе информации.
Регулярные и нерегулярные (регулярные файлы задаются пользователем через интерфейс ОС)

Pipe (именованный канал) - один из методов межпроцессного взаимодействия, для обмена информации между процессами.

Процесс - это программа, которая выполняется в данный момент, сама программа это набор инструкций, процесс же это непосредственное выполнение этих инструкций. Код программы, счётчик, адресное пространство, ресурсы.

ОС - это комплекс взаимосвязанных программ, интерфейсов, предназначенных для управления ресурсами компьютера, контроля работы программ и организации взаимодействия с пользователем. Запускается с pid 0 порождая pid 1. Далее pid 0 заканчивается. 

Fork (ответвление) - использует код текущего процесса(программы выполняемой в данный момент) и его счётчик как базу для создания параллельно выполняемого нового процесса в новом адресном пространстве. 

System V ips - механизм обмена данными между приложениями. 
- Разделяемая память shm
- Семафоры sem 
- Очереди сообщений msg

Очередь сообщений - представляет собой интерфейс, в котором один процесс может отправить сообщение или несколько в порядке очереди отправления другому процессу. Когда сообщение получено, оно удаляется из очереди. first in - first out.

- Семафоры - представляют собой интерфейс, в котором одни программы могут строго задавать свой порядок выполнения командами интерфейса, а так же контролировать "время" выполнения.

Семафор - это защищённая целочисленная неотрицательная переменная над которой совершаются атомарные операции: 
A(s,n) s+=n
D(s,n) while(s<n) blook; s=-n
Z(s,n) while(s!= 0) blook


Семафор Дейкстры -//- занять ресурсы(P операция)- блокирует семафор: while(s==0) blook else s = s-1, освободить ресурс (V операция)- деблокирует семафор: s=s+1.

- Разделяемая память(общяя память) - представляет собой мощный интерфейс, в котором программы могут использовать общую память для хранения и обмена информации, при этом программы могут работать так же и с собственными ресурсами памяти, недоступной другим процессам. 

Нить (поток выполнения)- интерфейс позволяющий делить выполнение процесса  на "нити" - единицы потоков для их поочередного выполнения на процессоре/процессорах, создавая тем самым параллельную работу. Они разделяют и используют совместно ресурсы процесса такие как память и даже код.
